# チューリングマシンを PHP で動かす

コンピュータの中では、いろいろな処理が複雑に見えます。  
でも、理論としての計算をかなり単純化すると、結局は「記号を読んで、書き換えて、次の状態へ進む」だけになります。

今回は、その最小モデルのひとつであるチューリングマシンを、PHPでそのまま読める形で実装します。  
教材としては、単項加算 `111+11 -> 11111` だけを扱います。

## チューリングマシンとは

チューリングマシンは、1936年にアラン・チューリングが考えた計算モデルです。  
今のCPUやメモリの直接の実装そのものではありませんが、「計算とは何か」を考えるための、とても強い土台になっています。

Webエンジニアの目線で言うと、これは「プログラムの処理を、かなり冷たく単純化したもの」です。  
入力を読み、状態を持ち、記号を書き換え、停止する。  
たったそれだけですが、理論上はかなり広い計算を表現できます。

## 形式的定義

チューリングマシンは、次の7つ組で表します。

`(Q, Γ, b, Σ, δ, q0, F)`

それぞれを、今回のPHP実装に対応づけるとこうなります。

| 記号 | 意味 | PHP側の対応 |
| --- | --- | --- |
| `Q` | 状態の集合 | `$states` |
| `Γ` | テープ記号の集合 | `$alphabet` |
| `b` | 空白記号 | `$blankSymbol` |
| `Σ` | 入力記号の集合 | `$inputAlphabet` |
| `δ` | 遷移関数 | `$delta` |
| `q0` | 初期状態 | `$initialState` |
| `F` | 受理状態の集合 | `$finalStates` |

この対応をそのままコードに落とすと、定義と実装の距離がかなり短くなります。  
教材では、この「対応が見えること」が大事です。

### コードの役割分担

実装の見通しをよくするため、クラスごとの役割も整理しておきます。

| クラス | 役割 | 代表メソッド |
| --- | --- | --- |
| `Tape` | テープの読み書き | `read()`, `write()`, `toString()`, `toWord()` |
| `TuringMachine` | 7つ組と実行ループ | `run()` |

`Tape` はデータ構造、`TuringMachine` は制御です。  
この分け方にすると、単項加算以外の機械にも流用しやすくなります。

## テープと状態

### テープ

テープは、左右に無限に伸びるメモリだと考えます。  
PHPでは、配列のキーに整数を使えば、負の位置も自然に扱えます。

`turing_machine/turing_machine.php` では、`Tape` クラスが担当しています。

### 状態

状態は、今どの手順にいるかを表します。  
たとえば今回の単項加算では、次のような状態を使います。

- `q0` : `+` を見つけるまで右へ進む
- `q1` : 右端まで進む
- `q2` : 末尾の `1` をひとつ消す
- `halt` : 停止

## 単項加算マシン

単項表記では、数字を `1` の個数で表します。  
`3 + 2` は `111 + 11` です。

この表記の利点は、加算が「記号の数を足すだけ」になることです。  
チューリングマシンとしては、かなり分かりやすい題材です。

### 状態遷移図

まずは全体の流れを、図っぽく眺めます。

```text
q0 --1/1,R--> q0
q0 --+/1,R--> q1
q1 --1/1,R--> q1
q1 --_/_,L--> q2
q2 --1/_,N--> halt
```

右方向へ進んで `+` を `1` に変え、最後の `1` をひとつ消すだけです。  
この単純さが、チューリングマシンの教材としてちょうど良いところです。

### 遷移表

単項加算の遷移は、次のように作れます。

| 状態 | 読んだ記号 | 書く記号 | 移動 | 次の状態 |
| --- | --- | --- | --- | --- |
| `q0` | `1` | `1` | `R` | `q0` |
| `q0` | `+` | `1` | `R` | `q1` |
| `q1` | `1` | `1` | `R` | `q1` |
| `q1` | `_` | `_` | `L` | `q2` |
| `q2` | `1` | `_` | `N` | `halt` |

やっていることは単純です。

1. `+` を見つけるまで右へ進む
2. `+` を `1` に変えて、右側の入力へ進む
3. 右端の空白まで進む
4. 最後の `1` をひとつ消す

これで、`111+11` は `111111` に変わったあと、最後の `1` をひとつ削って `11111` になります。

### トレースの見方

実行すると、各ステップで現在状態とテープが表示されます。

```text
step  0  state=q0  tape=[1] 1 1 + 1 1
step  1  state=q0  read=1 write=1 move=R next=q0  tape=1 [1] 1 + 1 1
step  2  state=q0  read=1 write=1 move=R next=q0  tape=1 1 [1] + 1 1
step  3  state=q0  read=+ write=1 move=R next=q1  tape=1 1 1 [1] 1 1
step  4  state=q1  read=1 write=1 move=R next=q1  tape=1 1 1 1 [1] 1
...
step  8  state=halt  read=1 write=_ move=N next=halt  tape=1 1 1 1 1 [_]
結果: 11111 (3 + 2 = 5)
```

この出力を追うと、計算が「頭の中の算数」ではなく、「記号の書き換えの連続」だと分かります。

## 実行方法

ファイルを直接実行します。

```bash
php turing_machine/turing_machine.php
```

引数を渡せば、別の入力でも試せます。

```bash
php turing_machine/turing_machine.php "1+1111"
php turing_machine/turing_machine.php "+111"
php turing_machine/turing_machine.php "1+1"
```

入力は `1` と `+` だけを受け付けます。  
`+` はちょうど1個必要です。

検証用の小さなPHPスクリプトも付けています。

```bash
php turing_machine/test_turing_machine.php
```

これで代表ケースと不正入力の確認ができます。

## 発展課題

ここまで理解できたら、次の題材にも挑戦できます。

- 単項乗算
- 回文判定
- ビジービーバー
- 停止性問題

特に停止性問題は重要です。  
「このプログラムは止まるか」を一般に判定できない、という話です。  
チューリングマシンを知ると、この話が机上の空論ではなくなります。

## まとめ

チューリングマシンは、計算をとても素朴な形に分解したモデルです。  
そして、その素朴さのおかげで、計算の本質が見えやすくなります。

今回のPHP実装は、あえて抽象度を上げすぎず、7つ組とコードの対応が見えるように作ってあります。  
まずは動かして、トレースを見て、記号の書き換えとしての計算に慣れるのがよいです。
