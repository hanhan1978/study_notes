# チューリングマシンのPHP実装(教材)

## Context

このリポジトリはWebエンジニア向けのC言語勉強会資料(markdown)を置いている。同じノリの教育用教材として、チューリングマシンをPHPで実装する。形式は「解説markdown + 動かして読める完成版PHPコード」のセット。サンプル機械は**単項加算**(`111+11` → `11111`)のみ。リポジトリにはまだコードが一切ないので、依存なし・単一ファイルの読みやすい構成で作る。

## 作成するファイル

```
003_turing_machine.markdown        # 解説資料(ルート直下、001_make / 002_debug の連番に合わせる)
turing_machine/turing_machine.php  # TM本体 + 単項加算のサンプル機械(依存なし、php コマンドだけで動く)
```

## 1. `turing_machine/turing_machine.php`

教育目的なので「形式的定義の7つ組がそのままコードに見える」ことを最優先にする。PHP 8系の素直な文法のみ(enum等の新しめの機能は避け、誰でも読める書き方)。コメントは日本語。

### 構成(クラス2つ + 実行部)

- **`Tape` クラス** — 無限テープ
  - 中身は `array<int,string>`(連想配列)+ 空白記号 `_`。負のインデックスも自然に扱えるので「左右に無限」を簡単に表現できる
  - `read(int $pos)`, `write(int $pos, string $symbol)`, `toString(int $headPos)`(ヘッド位置を `[1]` のように囲んで表示)

- **`TuringMachine` クラス** — 7つ組 (Q, Γ, b, Σ, δ, q0, F) をコンストラクタ引数にそのまま対応させる
  - 遷移関数 δ は `[状態][読んだ記号] => [書く記号, 移動(L/R/N), 次の状態]` の連想配列
  - `run(string $input, bool $trace = true)`:
    - 入力文字列をテープに展開し、ヘッド位置0・初期状態からループ
    - 毎ステップ `step / 現在状態 / テープ(ヘッド位置つき)` をトレース出力 — 動作が目で追えるのが教材の肝
    - 受理状態に入ったら停止して最終テープ内容を返す。遷移が未定義なら「拒否」、無限ループ対策に最大ステップ数(例: 10000)でガード

- **実行部(ファイル末尾)**
  - 単項加算マシンの遷移表を定義: 状態 `q0`(右へスキャンし `+` を `1` に書き換え)→ `q1`(右端まで進む)→ `q2`(末尾の `1` を1個消す)→ `halt`
  - `php turing_machine.php` で `111+11` を実行、`php turing_machine.php "1+1111"` のように引数でも入力を渡せる
  - 入力バリデーション(`1` と `+` 以外はエラー)

### 期待される実行イメージ

```
$ php turing_machine.php
step  0  state=q0  [1] 1  1  +  1  1
step  1  state=q0   1 [1] 1  +  1  1
...
step 13  state=halt  1  1  1  1  1
結果: 11111 (3 + 2 = 5)
```

## 2. `003_turing_machine.markdown`(解説資料)

既存の c_study 資料と同じ調子(です・ます調、Webエンジニア向け、手を動かす形式)で書く。構成:

1. **チューリングマシンとは** — 1936年チューリング、「計算」を定義するための最小の機械モデル。CPUとメモリの理論的な祖先、という導入
2. **形式的定義(7つ組)** — Q, Γ, b, Σ, δ, q0, F をそれぞれ平易に説明し、PHPコードのどこに対応するかを対応表で示す
3. **単項加算マシンを動かす** — 遷移表を載せ、`111+11` が `11111` になる過程をトレース出力つきで解説。「計算とは記号の書き換えである」がメッセージ
4. **実行方法** — `php turing_machine/turing_machine.php` と引数の渡し方
5. **発展課題** — 単項乗算、回文判定、ビジービーバー、停止性問題への言及(実装は載せず、考えるネタとして提示)

## 検証方法

- `php turing_machine/turing_machine.php` を実行し、トレースが出て最終結果が `11111` になること
- 引数で `1+1`(→ `11`)、`+111`(0 + 3 → `111`)などの境界ケースを実行して確認
- 不正入力(`abc` など)でエラーメッセージが出ること
- `php -l` で構文チェック
